### Central Processing Unit (CPU)

Composed of two main units: 
1. CU (control unit)
Directs operations of the processor. Responsible for the fetch-decode-execute cycle, managing all three operations, directing the computer's memory, ALU and I/O devices to response appropriately. 

2. ALU (Arithmetic and Logic unit)
Performs arithmetic and logic operations.
- Arithmetic operations: Subtraction, addition, multiplication, and division
- Logic operations: AND, OR, XOR and NOT

### Buses 
- Transfer data between various devices, including the CPU, memory, storage and peripherals. Buses have widths that are measured in bits. 

1. Control bus 
Transmit command and control signals from the CPU to other components of the system and vise versa. 

Since signals need to be sent and recived, these buses are `bidirectional`. Some signals that are transmitted through the control bus are read/write operations, interrupt requests, clock signals for synchronisation and status signals from hardware components. 

2. Data bus 
Carries data being processed through the CPU, memory and other peripherals. The width is imporant for determining how much data can be transfered at one time. 

Common data bus widths are 8, 16, 31 and 64 bits. As data needs to be read from and written to memory, data buses are usually `bidirectaional`.

3. Address Bus 
The address bus is used to transmit the address that is to be read from or written to in memory. 

The width determine the memory capacity of the system. For example a 32-bit address bus can address 2^32 memory locations. 

### Cores 

1. Single-core processors
The CPU has a single processing unit, processing only one task at a time. They are designed for simple tasks that do not require heavy multitasking.

Single core processors can run multiple applications, however as the CPU has to be shared between the applications, the performance is affected. 

2. Multi-core processors 
A CPU with multi-core processors have two or more cores that can run multiple instructions simultaneously. This includes of dual-core, quad-core, hexa-core, octa-core. Their perforamnce is significantly faster than single-core processors and ideal for multitasking, gaming and servers. 

However, extra software has to be written to take advantage of these extra cores. Older softwares that do not this would likely run at similar speeds as on a single core processor

3. Co-processors
A co-processor is a special type of processor whos main job is to support the main CPU. Tasks are offloaded by the CPU to the co-processors to run in parallel (parallel processing), to enhance system peformance. 

Examples of co-processors are Graphic processing units (GPU), Audio processors and digital signal processors (DSPs), which are used in telecommunications and image compression.

### Design philosophies 
CPUs are generally called 'genral-purpose processors', as they are able to handle a large range of tasks. They are degined to run the operating system, process user input and manage programs. CPUs are good at tasks where decisions need to be made quickly and where different types of work are being done at the same time. 

GPU are specialized co-processors that focus on specific types of task. They are made for processing large amounts of data in parallel. This means, they can work on many calculations in parallel. 


### Core architecture 
The CPU has only a few cores, but these cores are very powerful. Each core can handle many different intructions, but works optiamlly when executing one task at a time. Making the CPU good for running tasks such as the operating system, where quick responses are needed. CPUs also have features including branch prediction (Where the cpu tries to guess what will happen next), and out-of-order execution (where the CPU can work on tasks that are ready before tohers)

#### Techniqes of CPU program running

- Branch prediction: where the CPU makes predictions on what will happen next. If the prediction is correct, execution becomes faster. 

A CPU perfomrance-enhancing technique where the processor predicts the outcome of an instruction branch so that it can preload and execute instructions before the actual results is known

- Out-of-order execution: A CPU processing technique where instructions are executed as soon as the required resources and data are avalible, rather than stricktly following the original program sequence, to improve efficiency and processing speed.

#### Memory access and power efficiency 
The CPU and GPU access memory differently, the CPU uses a smaller high-speed memory cache to get data quickly. This is useful as the GPU only needs to access a small amount of data many times, such as running programs or handling user inputs. 

The GPU uses its own special memory called VRAM (video ram), that has a higher bandwidth, meaning it can move large amounts of data at once, such as iamges and videos. However, the GPU uses more power because it must process a lot of data at the same time, especially when rendering videos or running complex simulations.

#### CPU and GPU working together ot increase video-game performance

For gaming, CPU and GPU work in union for seamless and immersive experiences. 
- The CPU handles the game's core logic, including rules, physical calculations and AI behaviour. It processes the input from the player (processing the outcome of thier actions and updating the game state accordingly).

The GPU's primary role to render the game's visuals. It processes vertex and pixel data to draw images on to the screen, including 3D objects, textures and effects such as lighting and shadows.

##### Typical scenario

1. **Player input:** The player presses a key to move a character. The CPU processes this input, updates the character’s position based on game physics and determines the new game state.

2. **Data preparation:** The CPU prepares the new position and state data and sends it to the GPU.

3. **Rendering:** The GPU updates the frame with the character’s new position, applies lighting and shading and renders the scene.

4. **Display:** The rendered frame is displayed on the screen, providing immediate feedback to the player.
